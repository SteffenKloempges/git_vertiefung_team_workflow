Heute lernen wir Git & GitHub etwas besser kennen

Was für Schritte kennen wir bereits ?

  1. git init ==> erstellt ein lokales git repo
  2. Wie bereiten wir Dateien vor ?
    • git add . ==> bereitet alle Dateien/Änderungen für die Staging Area vor
    • git add "datei.name" ==> bereitet eine einzelne Datei/Änderung für die Staging Area vor
  3. git commit -m "message / hier bitte etwas aussagekräfiges" ==> speichert die stating area im lokalen git repo und erstellt einen sogenannten "snapshot"
  4. git remote add origin "git@github.com:LinkZumGitHubRepo" ==> erstellen eine verbindung zwischen dem lokalen git repo und dem remote repo auf github
  5. git branch -M main ==> erstellt uns einen main branch auf dem wir dann arbeiten
  6. git push -u origin main ==> lädt lokales git repo in das remote repo hoch, in den main branch

!!! Schritt 4 bis 6 bekommt ihr von GitHub nach erstellen eines Repositories !!!

Was ist ein Branch ?

Ein Branch ist ein seperates Arbeitsverzeichnis in unserem Repository, es können mehrere branches in einem Repo existieren und unabhängig voneinander bearbeitet werden.

Was ist ein Repository ?

Repository = Depot/Lager, hier werden alle von uns hinzugefügen Dateien gespeichert, um sie später gebündelt an einem Platz zu haben.

Was ist die Stage/Staging Area ?

Ist wie ein Wartezimmer beim Arzt, hier werden die von uns geänderten Dateien vorbereitet um ins Repo zugelangen.

Weitere Git-Befehle:

  • git log ==> zeigt uns daten über unsere commits/snapshots an
  • git branch ==> zeigt uns alle verfügbaren branches an und auf welchem wir uns gerade befinden
  • git checkout "name" ==> lässt uns zwischen unseren lokalen branches wechseln
